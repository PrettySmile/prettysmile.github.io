# 分散式架構

將「應用系統」拆分為多個「**分佈在不同機器或地區的服務」**，透過「網路通信協作」完成任務。

透過多個獨立節點提供服務，提高擴展性和容錯能力。

## 說明
分散式系統（Distributed System）指的是由多個獨立計算機組成的系統，這些計算機通過網絡協作來完成共同的目標。這類系統廣泛應用於區塊鏈、微服務架構、雲計算、大數據處理等領域。

## **可用性（Availability）**

- 系統應該始終可用，即使某些節點故障或網絡出現問題，也應該能提供服務。
- 高可用架構常使用 **負載均衡（Load Balancing）** 和 **自動故障轉移（Failover）** 技術。
