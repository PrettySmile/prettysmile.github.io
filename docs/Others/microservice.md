# 微服務架構

**多個獨立可部署的服務（Service）**

每個服務可以**獨立開發、測試、部署**，不影響其他服務。

每個服務負責特定的業務功能，並透過 **API** 進行通信。

微服務之間通常使用 **RESTful API**（HTTP）或 **消息隊列（如 RabbitMQ、Kafka）** 來通信，避免了直接的依賴關係。

可擴展且需要高可用性的系統。

服務可獨立擴展

適合場景：大型系統(如Amazon)、高併發系統(支付、聊天、電商)、頻繁部署與迭代。

適合場景：需要高擴展性、靈活開發、多團隊協作。
